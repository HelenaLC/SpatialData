Advancing spatial omics technologies generate vast multimodal datasets, integrating high-resolution imaging with single-molecule transcriptomics. Managing this complexity requires a unified, technology-agnostic, and scalable data model. While Python’s spatialdata framework (Marconato et al., 2024) established a standard via the OME-Zarr specification, R users have lacked a native interface to leverage this ecosystem within Bioconductor.

We present SpatialData, an R package providing a comprehensive S4 interface to this framework. It implements the five core elements—multi-scale Images and Labels (raster), large-scale Points and Shapes (vector), and associated feature Tables—utilizing ZarrArray and DelayedArray for lazy on-disk raster processing and geoarrow for high-performance point manipulation. A central feature is its coordinate reference system (CRS) and transformation engine, enabling seamless alignment and querying across multiple coordinate spaces.

The significance of this work lies in the adoption of OME-Zarr as a standardized, cloud-ready on-disk format for multi-omic spatial data. This architecture enables a truly language-agnostic workflow: researchers can leverage Python’s state-of-the-art machine learning and image analysis tools while transitioning to R for advanced statistical modeling and spatial statistics. By bridging these two ecosystems, SpatialData provides the foundational infrastructure for multimodal analysis, ensuring data remains FAIR and accessible regardless of the preferred computational environment.
